DATABASE LAYER
  • NOT NULL constraint on projects.customer_id
  • Trigger: verify_customer_exists() - validates customer before insert/update
  • Trigger: prevent_customer_deletion_with_projects() - prevents deletion
  • Views: customer_project_summary, projects_with_customers
  • Indexes: For fast customer-project lookups

SERVICE LAYER
  Enhanced customerService.js:
    • validateCustomerExists(customerId) - Check if customer valid
    • getCustomerWithStats(customerId) - Customer + project stats
    • getCustomerProjectSummary() - All customers with stats
    • batchValidateCustomers(customerIds) - Bulk validation
    • Enhanced error messages

  Enhanced projectService.js:
    • createProject() REQUIRES customer_id (enforced)
    • getProjectsByCustomer(customerId) - Filter by customer
    • getProjectsWithCustomers() - Joins with customer data
    • getProjectWithCustomer(id) - Single project + customer
    • batchCreateProjects(customerId, projects) - Bulk create

UI LAYER
  CustomerSelector Component:
    • Search existing customers
    • Create new customer button
    • Email validation
    • Error handling
    • Required field indicator

  CustomerCreationModal Component:
    • Full customer form
    • Address fields
    • Phone, email, notes
    • Success confirmation
    • Input validation

  ProjectCreationGuard Component:
    • Checks: Do customers exist?
    • If NO: Redirect with helpful message
    • If YES: Allow project creation
    • Prevents orphaned projects
