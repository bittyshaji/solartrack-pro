# CloudCatholic Pro - Modern Ecommerce Platform
## Complete Architecture & Implementation Guide

**Project**: Modern ecommerce platform for Catholic products (India-focused)
**Status**: MVP Planning Phase
**Target Launch**: Production-ready within 4-6 weeks
**Date**: March 26, 2026

---

## 🎯 EXECUTIVE SUMMARY

We're building a lean, scalable MVP for selling Catholic products (books, magazines, supplies, gifts) to Indian customers. The architecture prioritizes:
- **Fast time-to-market** (MVP in 2-3 weeks)
- **Zero upfront costs** (free-tier tools only)
- **Real order processing** (Razorpay integration)
- **Easy data export** (future Shopify/WooCommerce migration)
- **Mobile-first UX** (60% of Indian ecommerce is mobile)

## 🏗️ PART 1: TECH STACK DECISION

### Selected Stack

| Layer | Technology | Why This Choice |
|-------|------------|-----------------|
| **Frontend** | Next.js 14 (React) | SSR for SEO, built-in API routes, Vercel deployment, excellent ecommerce patterns |
| **Backend** | Next.js API Routes + Supabase | Serverless, scales to zero cost, PostgreSQL reliability |
| **Database** | PostgreSQL (Supabase) | Relational, perfect for products/orders, easy CSV export for migration |
| **Auth** | Supabase Auth (for Phase 2) | Free, JWT-based, integrates seamlessly |
| **Payments** | Razorpay | #1 payment gateway in India, UPI support, developer-friendly |
| **File Storage** | Supabase Storage | Free tier: 1GB, integrated with database |
| **Hosting** | Vercel (frontend) | Next.js native, free tier, auto-scaling, 0ms cold starts |
| **Analytics** | Supabase + Vercel built-in | Free, privacy-friendly, no extra cost |
| **Email** | Resend (free tier) | 100 emails/day free, perfect for transactional emails |

### Why Not Alternatives?

- ❌ **Shopify**: Costs $29/month + 2% commission before MVP validation
- ❌ **WooCommerce**: Requires managed hosting, more complexity
- ❌ **Firebase**: Firestore pricing can spike; PostgreSQL is better for relational data
- ❌ **Stripe**: Not ideal for India; Razorpay has better UPI/local payment integration
- ✅ **This stack**: $0 until you generate revenue, 100% scalable

## 🗄️ PART 2: DATABASE SCHEMA

### Core Tables (Relational)

```sql
-- 1. PRODUCTS TABLE
CREATE TABLE products (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  category_id BIGINT NOT NULL REFERENCES categories(id) ON DELETE CASCADE,
  name VARCHAR(255) NOT NULL,
  slug VARCHAR(255) UNIQUE NOT NULL,
  description TEXT,
  price DECIMAL(10, 2) NOT NULL,
  currency VARCHAR(3) DEFAULT 'INR',
  image_url VARCHAR(500),
  stock_quantity INT DEFAULT 0,
  sku VARCHAR(100) UNIQUE,
  rating DECIMAL(3, 2) DEFAULT 0,
  review_count INT DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  -- Indexes for fast queries
  INDEX idx_category ON products(category_id),
  INDEX idx_slug ON products(slug),
  INDEX idx_active ON products(is_active)
);

-- 2. CATEGORIES TABLE
CREATE TABLE categories (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name VARCHAR(100) NOT NULL UNIQUE,
  slug VARCHAR(100) UNIQUE NOT NULL,
  description TEXT,
  image_url VARCHAR(500),
  display_order INT DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  INDEX idx_slug ON categories(slug),
  INDEX idx_active ON categories(is_active)
);

-- 3. CUSTOMERS TABLE (Phase 2: Auth)
CREATE TABLE customers (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255),  -- Nullable for MVP (unauthenticated checkout)
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  phone VARCHAR(20),
  default_address_id BIGINT REFERENCES addresses(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  INDEX idx_email ON customers(email)
);

-- 4. ADDRESSES TABLE
CREATE TABLE addresses (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  customer_id BIGINT REFERENCES customers(id) ON DELETE CASCADE,
  type VARCHAR(20) DEFAULT 'shipping',  -- 'shipping' or 'billing'
  name VARCHAR(100) NOT NULL,
  phone VARCHAR(20) NOT NULL,
  street_address VARCHAR(255) NOT NULL,
  city VARCHAR(100) NOT NULL,
  state VARCHAR(100) NOT NULL,
  postal_code VARCHAR(20) NOT NULL,
  country VARCHAR(100) DEFAULT 'India',
  is_default BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  INDEX idx_customer ON addresses(customer_id)
);

-- 5. ORDERS TABLE
CREATE TABLE orders (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  order_number VARCHAR(50) UNIQUE NOT NULL,  -- "ORD-2026-000001"
  customer_id BIGINT REFERENCES customers(id) ON DELETE SET NULL,
  customer_name VARCHAR(100) NOT NULL,
  customer_email VARCHAR(255) NOT NULL,
  customer_phone VARCHAR(20) NOT NULL,
  shipping_address_id BIGINT REFERENCES addresses(id) ON DELETE SET NULL,

  -- Order summary
  subtotal DECIMAL(12, 2) NOT NULL,
  tax_amount DECIMAL(12, 2) DEFAULT 0,
  shipping_cost DECIMAL(12, 2) DEFAULT 0,
  discount_amount DECIMAL(12, 2) DEFAULT 0,
  total_amount DECIMAL(12, 2) NOT NULL,
  currency VARCHAR(3) DEFAULT 'INR',

  -- Status tracking
  status VARCHAR(50) DEFAULT 'pending',  -- 'pending', 'confirmed', 'shipped', 'delivered', 'cancelled'
  payment_status VARCHAR(50) DEFAULT 'unpaid',  -- 'unpaid', 'paid', 'failed'
  payment_method VARCHAR(50) DEFAULT 'razorpay',
  razorpay_order_id VARCHAR(100),
  razorpay_payment_id VARCHAR(100),

  -- Timestamps
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  confirmed_at TIMESTAMP WITH TIME ZONE,
  shipped_at TIMESTAMP WITH TIME ZONE,
  delivered_at TIMESTAMP WITH TIME ZONE,
  cancelled_at TIMESTAMP WITH TIME ZONE,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  -- Indexes
  INDEX idx_order_number ON orders(order_number),
  INDEX idx_customer ON orders(customer_id),
  INDEX idx_status ON orders(status),
  INDEX idx_payment_status ON orders(payment_status),
  INDEX idx_created ON orders(created_at DESC)
);

-- 6. ORDER_ITEMS TABLE
CREATE TABLE order_items (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  order_id BIGINT NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id BIGINT NOT NULL REFERENCES products(id),
  product_name VARCHAR(255) NOT NULL,  -- Snapshot
  product_price DECIMAL(10, 2) NOT NULL,  -- Snapshot (price at time of order)
  quantity INT NOT NULL,
  line_total DECIMAL(12, 2) NOT NULL,  -- quantity * product_price
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

  INDEX idx_order ON order_items(order_id),
  INDEX idx_product ON order_items(product_id)
);

-- 7. CART ITEMS TABLE (Temporary, for session-based cart)
CREATE TABLE cart_items (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  session_id VARCHAR(100) NOT NULL,  -- Session token, not user-specific (for MVP)
  product_id BIGINT NOT NULL REFERENCES products(id) ON DELETE CASCADE,
  quantity INT NOT NULL,
  added_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  expires_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() + INTERVAL '7 days',

  INDEX idx_session ON cart_items(session_id),
  UNIQUE (session_id, product_id)
);
```

### Key Design Decisions

1. **No User Authentication in MVP**
   - MVP allows unauthenticated checkout (guest orders)
   - Customer data stored in `orders` table (name, email, phone)
   - Phase 2: Add `customers` table for returning customers

2. **Product Snapshots**
   - `order_items` stores product_name + product_price at time of order
   - If product is deleted, order history still accurate

3. **Indexing Strategy**
   - Indexed all foreign keys for join performance
   - Indexed frequently filtered columns (status, category, slug)
   - Indexed timestamps for sorting

4. **Easy CSV Export**
   - Flat structure, no deeply nested relationships
   - All orders can be exported as: `SELECT * FROM orders JOIN order_items`
   - Easy to migrate to Shopify/WooCommerce

## 💻 PART 5: KEY CODE PATTERNS

### API Route Pattern (Next.js)

```typescript
// app/api/products/route.ts
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);

export async function GET(request: Request) {
  try {
    const { searchParams } = new URL(request.url);
    const category = searchParams.get('category');

    let query = supabase
      .from('products')
      .select('*')
      .eq('is_active', true);

    if (category) {
      query = query.eq('category_id', category);
    }

    const { data, error } = await query;

    if (error) throw error;

    return Response.json({ success: true, data });
  } catch (error) {
    return Response.json(
      { success: false, error: error.message },
      { status: 500 }
    );
  }
}
```

### Cart Management (Client-side)

```typescript
// lib/cart.ts
export const getCart = (): CartItem[] => {
  if (typeof window === 'undefined') return [];
  const cart = localStorage.getItem('cart');
  return cart ? JSON.parse(cart) : [];
};

export const addToCart = (productId: number, quantity: number) => {
  const cart = getCart();
  const existing = cart.find(item => item.product_id === productId);

  if (existing) {
    existing.quantity += quantity;
  } else {
    cart.push({ product_id: productId, quantity });
  }

  localStorage.setItem('cart', JSON.stringify(cart));
};

export const removeFromCart = (productId: number) => {
  const cart = getCart().filter(item => item.product_id !== productId);
  localStorage.setItem('cart', JSON.stringify(cart));
};
```

### Razorpay Integration

```typescript
// app/api/payments/razorpay-create/route.ts
import Razorpay from 'razorpay';

const razorpay = new Razorpay({
  key_id: process.env.RAZORPAY_KEY_ID!,
  key_secret: process.env.RAZORPAY_KEY_SECRET!,
});

export async function POST(request: Request) {
  const { totalAmount, orderId } = await request.json();

  try {
    const order = await razorpay.orders.create({
      amount: Math.round(totalAmount * 100), // Convert to paise
      currency: 'INR',
      receipt: orderId,
      payment_capture: 1,
    });

    return Response.json({ success: true, order });
  } catch (error) {
    return Response.json(
      { success: false, error: error.message },
      { status: 500 }
    );
  }
}
```

## 🚀 PART 7: DEPLOYMENT GUIDE

### Step 1: Setup Supabase (Database)

1. Go to [supabase.com](https://supabase.com)
2. Create new project (free tier)
3. Get URL and keys from Settings → API
4. Run SQL schema (see database section)

### Step 2: Setup Razorpay (Payments)

1. Go to [razorpay.com](https://razorpay.com)
2. Create merchant account
3. Get API keys from Settings
4. Enable UPI in payment methods

### Step 3: Setup Vercel (Hosting)

1. Go to [vercel.com](https://vercel.com)
2. Import your Next.js repo from GitHub
3. Add environment variables
4. Deploy (auto on every push)

### Step 4: Custom Domain (Optional)

1. Buy domain on Namecheap/GoDaddy
2. Point DNS to Vercel nameservers
3. Add domain in Vercel project settings
