WHAT THE SCRIPT DOES (14 Steps)
================================================================================

STEPS 1-3: Table Structure
  - Removes old project_id and status columns from project_customers
  - Adds customer fields: name, email, phone, address, city, state, postal_code
  - Adds company, notes, is_active, created_at, updated_at timestamps
  - Adds customer_id column to projects table

STEP 4-5: Legacy Migration
  - Creates "CUST-LEGACY-0000" customer for orphaned projects
  - Migrates all projects with NULL customer_id to legacy customer

STEP 6: Data Integrity
  - Applies NOT NULL constraint on customer_id (prevents future orphans)

STEPS 7-10: Validation & Safety
  - Creates trigger: Validates customer exists before project creation
  - Creates trigger: Prevents deactivating customers with active projects
  - Ensures referential integrity

STEP 11: Performance
  - Creates 4 indexes on customer_id, is_active columns
  - Speeds up customer lookups and filtering

STEPS 12-14: Reporting
  - Creates 2 views:
    • customer_project_summary (customer analytics)
    • projects_with_customers (joined view for queries)
  - Creates monitoring function: get_customer_project_stats()

================================================================================
TROUBLESHOOTING
================================================================================

ERROR: "column already exists"
  → Safe to ignore - script uses ADD COLUMN IF NOT EXISTS
  → Means column was already in table from prior run

ERROR: "cannot find table"
  → Check that tables exist: project_customers, projects
  → Verify you're in correct schema (public)

ERROR: "function already exists" or "trigger already exists"
  → Safe to ignore - script drops and recreates them
  → Ensures latest version is installed

ERROR: "cannot deactivate customer with active projects"
  → Expected behavior - referential integrity working
  → Cancel projects before deactivating customer

TIMEOUT (>30 seconds)
  → Check Supabase dashboard status
  → Retry execution
  → Contact support if persistent
