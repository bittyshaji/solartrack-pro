VERIFICATION QUERIES (Run after execution)
═══════════════════════════════════════════════════════════════════════════════

After the schema is applied, verify the installation by running these queries
in the Supabase SQL Editor:

QUERY 1: Check NOT NULL Constraint
───────────────────────────────────
SELECT is_nullable 
FROM information_schema.columns
WHERE table_name = 'projects' AND column_name = 'customer_id';

Expected Result:
  is_nullable = "NO"  ✅

QUERY 2: Check Triggers (Should be 2)
──────────────────────────────────────
SELECT trigger_name, event_object_table
FROM information_schema.triggers
WHERE event_object_table IN ('projects', 'project_customers')
ORDER BY trigger_name;

Expected Results:
  ✅ prevent_customer_deletion
  ✅ projects_customer_validation

QUERY 3: Check Indexes (Should be 4)
──────────────────────────────────────
SELECT indexname, tablename
FROM pg_indexes
WHERE tablename IN ('projects', 'project_customers')
AND indexname NOT LIKE 'pg_%'
ORDER BY indexname;

Expected Results:
  ✅ idx_project_customers_active_name
  ✅ idx_project_customers_is_active
  ✅ idx_projects_customer_id
  ✅ idx_projects_customer_status

QUERY 4: Check Views (Should be 2)
───────────────────────────────────
SELECT table_name
FROM information_schema.views
WHERE table_schema = 'public'
AND table_name IN ('customer_project_summary', 'projects_with_customers')
ORDER BY table_name;

Expected Results:
  ✅ customer_project_summary
  ✅ projects_with_customers

QUERY 5: Check Functions (Should be 3)
──────────────────────────────────────
SELECT routine_name
FROM information_schema.routines
WHERE routine_schema = 'public'
AND routine_name IN (
  'verify_customer_exists',
  'prevent_customer_deletion_with_projects',
  'get_customer_project_stats'
)
ORDER BY routine_name;

Expected Results:
  ✅ get_customer_project_stats
  ✅ prevent_customer_deletion_with_projects
  ✅ verify_customer_exists

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

ERROR: "Relation 'projects' does not exist"
CAUSE: Database schema not initialized yet
FIX: Run initial database setup before Phase 1

ERROR: "Constraint projects_customer_id_not_null already exists"
CAUSE: Constraint was already applied
FIX: This is OK - means it's already in place. Just continue.

ERROR: "Trigger 'projects_customer_validation' already exists"  
CAUSE: Trigger was already created
FIX: Script handles this with DROP IF EXISTS - safe to re-run

ERROR: "Cannot cast column 'customer_id' to NOT NULL - existing NULL values"
CAUSE: Some projects have NULL customer_id
FIX: Update NULL values first:
  UPDATE projects
  SET customer_id = (SELECT customer_id FROM project_customers LIMIT 1)
  WHERE customer_id IS NULL;

ERROR: "Permission denied"
CAUSE: Using key with insufficient permissions
FIX: Use Supabase Dashboard - you have full access there
